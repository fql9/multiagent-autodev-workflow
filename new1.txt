## 2) “能否严格执行？”——需要分两层回答

### 2.1 Claude Code 对“加载这些配置文件”的执行是很严格的（✅ 可严格）
只要文件位置/格式正确，Claude Code 会按官方机制加载：
- 记忆（CLAUDE.md）启动加载、递归查找、支持 `@import`。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/memory))  
- subagents 从 `.claude/agents/` 发现，并可自动委派或显式点名调用。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/subagents))  
- skills 从 `.claude/skills/` 自动发现并在相关任务中使用。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/skills))  
- commands 从 `.claude/commands/` 加载。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/slash-commands?utm_source=openai))  
- hooks/permissions 从 `.claude/settings.json` 读取并生效。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/hooks?utm_source=openai))  

### 2.2 但“让大模型乖乖听话”这种行为层面的“严格”，只能做到**工程上尽量强约束**，无法数学意义 100% 保证（⚠️）
原因是：
- **自动 subagent 选择**是基于 `description` 匹配 + 当前上下文的智能委派，不是规则引擎硬匹配；官方也明确可以“自动委派/显式调用”两种方式。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/subagents))  
- **skills 是否触发**是模型自主决定（官方叫 model-invoked），不是每次必触发。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/skills))  
- 你 README 里提到的“Dashboard、DAG、质量门（pre-commit/pre-merge）”——这些并不是 Claude Code 原生特性，必须通过：
  - `/swe` 这类命令把步骤写死（流程化提示词），或  
  - hooks 在关键事件上强制检查/阻断（比如 PreToolUse 阻止不合规的 Bash/Write），([docs.claude.com](https://docs.claude.com/en/docs/claude-code/hooks-guide?utm_source=openai))  
  - permissions 做“最小权限”硬限制（deny/ask/allow）。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/settings?utm_source=openai))  
才能接近“严格”。

## 3) 你这个框架里，最可能“看起来有、但不一定严格生效”的点（需要你核对文件内容）

基于 README 的描述，我建议你重点自查这几类“常见踩坑”：

1) **subagent frontmatter 是否完全符合官方字段名与格式**  
官方示例是 YAML frontmatter（`name/description/tools/model/permissionMode/skills`）。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/subagents))  
- `tools` 是“逗号分隔”还是 YAML list，都要确保 Claude Code 能解析（官方示例里两种形式都有出现：Markdown 文件示例多为逗号分隔；CLI JSON 是数组）。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/subagents))  
- `permissionMode` 的取值必须在：`default/acceptEdits/bypassPermissions/plan/ignore`。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/subagents))  

2) **skills 的 `SKILL.md` 是否带 `name` + `description`**  
`description` 是 Claude 决定“何时使用该 Skill”的关键。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/skills))  

3) **commands 是否有 `description` frontmatter（如果你希望 Claude 能自动调用命令）**  
否则只能“人手动输入 /swe”，Claude 不一定会通过 `SlashCommand` 工具自动触发它。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/slash-commands?utm_source=openai))  

4) **你说的“质量门 / Dashboard 更新 / 高风险操作审批”是否真的用 hooks/permissions 落地**  
- hooks 可以在 `PreToolUse` 阻断工具调用并给 Claude 反馈。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/hooks-guide?utm_source=openai))  
- permissions 可以对 Bash/Write 等做 allow/deny/ask。([docs.claude.com](https://docs.claude.com/en/docs/claude-code/settings?utm_source=openai))  
如果 `.claude/settings.json` 里没有这些规则，那么“严格执行”基本只能靠提示词自觉，稳定性会差很多。
