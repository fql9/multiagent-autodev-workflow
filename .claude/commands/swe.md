# /swe 命令 - 软件工程任务执行器

执行一个完整的软件工程任务，包括分析、规划、实现、测试和审查。

## 使用方式
```
/swe <任务描述>
```

## 执行流程

### 1. 需求分析
- 理解用户任务目标
- 提取关键需求和约束
- 生成验收标准

### 2. 代码分析
调用 `repo-scout` 子智能体：
- 搜索相关代码
- 分析依赖关系
- 识别影响范围

### 3. 任务规划
生成任务 DAG：
- 分解为子任务
- 确定依赖关系
- 分配执行者

### 4. 方案设计（如需要）
调用 `architect` 子智能体：
- 设计技术方案
- 定义接口契约
- 评估风险

### 5. 代码实现
调用 `implementer` 子智能体：
- 按计划编写代码
- 生成 patch
- 更新测试

### 6. 验证
调用 `tester` 子智能体：
- 运行测试
- 静态检查
- 生成报告

### 7. 审查
调用 `reviewer` 子智能体：
- 代码审查
- 验收检查
- 最终确认

### 8. 状态更新
更新 DASHBOARD.md：
- 记录完成状态
- 汇总产出物
- 记录审计日志

## 参数
- `$ARGUMENTS`: 任务描述

## 示例
```
/swe 修复用户登录时的空指针异常
/swe 添加用户邮箱验证功能
/swe 重构 UserService 提高可测试性
```
